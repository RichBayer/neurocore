# SYSTEM LAYERS

NeuroCore is composed of the following logical layers:

---

## 1. Distribution Layer (NEW)

This layer defines how NeuroCore is packaged and exposed to users.

Examples:

- Argus (system intelligence distribution)
- Future distributions (automation, home, etc.)

This layer does NOT modify the runtime.

It provides:

- entrypoint behavior
- feature constraints
- user-facing experience

---

## 2. Interface Layer

Entry points into the system.

Examples:

- CLI (`ai`)
- Argus CLI (future wrapper)
- future UI / voice interfaces

---

## 3. Runtime Layer (Core System)

Persistent system responsible for all processing.

Components:

- NeuroCore Daemon
- Runtime Manager
- Control Plane

All requests MUST pass through this layer.

---

## 4. Execution Layer

Handles controlled system interaction.

Components:

- Execution Engine
- Tool Registry
- Tool Interface

Execution Path:
control_plane → execution_engine → tool

---

## 5. Logic Layer

Responsible for reasoning and routing.

Components:

- Router (query rewriting, intent handling)

Constraints:

- cannot execute commands
- must obey control plane

---

## 6. Knowledge Layer

Handles retrieval and context.

Components:

- Chroma vector database
- embedding model
- retrieval system

---

## 7. Model Layer

LLM runtime:

- Ollama (local models)

---

## 8. System Config Layer

Location:
 /mnt/g/ai/system/

Defines:

- personalities
- hardware profiles
- system state

---

# SYSTEM INVARIANT (UNCHANGED)

All execution flows through:

daemon → runtime_manager → control_plane

This rule is NEVER bypassed.

---

# KEY ADDITION

Argus is:

- NOT part of runtime
- NOT part of control plane
- NOT part of execution engine

Argus is:

> a distribution and interface constraint layer that defines how NeuroCore is used

---

# DESIGN PRINCIPLE

NeuroCore = Core Cognitive System  
Argus = Controlled System Intelligence Interface

---

# FUTURE EXPANSION MODEL

Additional distributions may be introduced:

- Argus (system intelligence)
- HomeCore (home automation)
- DevCore (development assistant)

All must:

- use the same runtime
- obey the same control plane
- share the same execution system

---

# FINAL RULE

The runtime is singular.

Distributions are interchangeable.

The system must never fragment into multiple runtimes.
